ANNOTATION
From the one hand the document should describe usage of NDN parser tool
(possibly separate user guide will be created in future or NDN parser
guide will be included into OKT-UG-0000111-TE_EL).

From the other hand the document should contain a design of the NDN parser
according to HLD template. Particularly following issues should be clarified:

* NDN parser modules and utilities (decomposition);

* NDN parser data flow and data organization (intermediate files, 
  internal structures, etc.);

* configurable features: 
  - customizing of output by user: 
      including pieces of source code, 
      user-defined types, 
      splitting to modules,
      generation of multi-language interfaces, etc.
  - specifiing of NDN parser work mode (NDN verification only, headers only, etc.);
  - level of debug.
