Test Packages

The document should contain results of research and list of proposals
related to TE test packages framework.

ANNOTATION
The document should contain:
  detailed definitions of tests and test packages;
  source organization description (including all naming conventions);
  sharing of internal libraries by test packages;
  initialization and shutdown of test packages;
  tests running issues (format of Test Subsystem configuration file;
                        Dispatcher and Test Subsystem command line options);
  possibility to run tests one-by-one, simultaneously or in the debugging mode;
  possibility to run tests checking specified requirement(s);
  specification and iteration of test parameters;
  dynamic building of test packages on behalf of Test Subsystem.
