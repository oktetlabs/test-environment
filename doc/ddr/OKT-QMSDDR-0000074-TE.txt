TITLE Proteos: guidelines for design of configuration trees

PURPOSE 
Provide recommendations to implementor of Test Agent how to name particular
objects and which relations exist between well-known objects.

AUDIENCE End users and TE project team-members

ACCESS OKTET employees and customers

TMPL OKT-TMPL-0000006

COMPLY
OKT-REQ-0000010-TE
OKT-ARCH-0000011-TE
OKT-HLD-0000069-TE_CS
OKT-HLD-0000071-TE_CS_DATA

ANNOTATION
This document is not strict specification, but rather a list of conventions
to be used by implementors of all Test Agents to allow test writers
create more portable tests.

It should describe objects, their values and childrens, which
should exist on most Test Agents: network interfaces, addresses, 
ARP records and routes, DNS and DHCP configuration parameters and so on.

It is assumed that this document will be updated by additional
descriptions after implementation new Test Agents and Test Packages.
