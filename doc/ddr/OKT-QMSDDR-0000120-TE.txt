TITLE
Proteos: traffic RCF application domain framework

PURPOSE
User's Guide (introductory reading) for customers and team-members, which want
to deal with traffic application domain of RCF.

AUDIENCE
End users and TE project team-members

AUDIENCE
OKTET employees and customers

ANNOTATION
This document should describe:
* terminology explanation;
* description of services provided by RCF application domain to users;
* introduction to ASN.1;
* ASN.1 textual conventions for CSAPs, patterns and templates;
* definitions for eth, ipv4.eth, udp.ipv4.eth, tcp.ipv4.etg, data.udp.ipv4.eth,
  data.tcp.ipv4.eth, atm and aal5.atm stacks (possible parameters of CSAPs,
  templates and patterns); possibility of incomplete stacks (like 
  udp.ipv4 - without eth) should be discussed as well;
* decomposition of RCF application domain software;
* libraries allowing user to dynamically parse/build templates and patterns;
* service provided by NDN parser to user.

DETAILS
Many examples should be provided. Strict ASN.1 definitions should be provided
for all mentioned stacks. Input and output of NDN parser should be described
in details.
